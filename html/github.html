Github Collaboration

What are Git and GitHub?

Are Git and GitHub the same? No, because: Git is a version control system (VCS), that “…tracks changes to files over a period of time.”(Kerr, 2024) While GitHub is “…a cloud-based platform where you can store share, and work together with others to write code.”(GitHub Docs, n.d.-b)

Put simply, Git is a tool for version control that tracks every change made in the file or code, and GitHub is a platform that is based on the technology that Git offers and “…lets you manage Git repositories”(Devmountain, n.d.) so it’s possible to collaborate and share.

What are the benefits of collaborating using Git and GitHub?

Many advantages are worth mentioning, but here are some of them:

  • Makes it possible to collaborate as a team on one file. Instead of creating a new copy of the file for each iteration, you can have one main file because Git tracks all the changes done over time. By doing this, the developer can see(GitHub Docs, n.d.-a):
  • View the entire timeline. Git allows developers to “…see the entire timeline of their changes, decisions, and progression of any project in one place.”(GitHub Docs, n.d.-a) This makes the collaboration transparent and reproducible(Coursera, 2023).
  • Real-time collaboration. It breaks down the communication barriers and makes it possible to have different working methods within the team (synchronously and physically/digitally, or/and asynchronously)(Coursera, 2023; GitHub Docs, n.d.-a).
  • Create branches. By creating branches from the main copy of files, developers can work without fearing that the changes will impact the work of the rest of the team. When the work is ready, it can be integrated into the main file.(GitHub Docs, n.d.-b)
  • Restore earlier versions. Should something happen, e.g. that the integrated code affects the program negatively, developers can easily revert to previous versions if necessary.(Coursera, 2023)
